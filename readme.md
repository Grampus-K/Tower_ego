
当你看到这份readme的时候，你可能要接手这份项目了，没事，先往下看吧
1、项目介绍：

这个项目是2024年陶杰老师接的一个铁塔的项目，那边要求我们在复杂环境下自主飞行，实际上就是要自主避障。而我的实现方式是直接复现ego planner（fast drone250）
2、额外工作：

    实际上原先的想法是用px4装RTK然后输出的里程计，用那个里程计来定位，因为原先的vins fusion鲁棒性实在是不高（垃圾vio）。所以我把px4ctrl整个文件全部舍弃了，换成了planner manager里面的那个px4_crtl_v2，里面的里程计不再用的是vinsfusion直接输出的里程计，而是px4输出的local odometry。这样就可以直接在qgc地面站里面改EKF2_AID_MASK那个参数，来使用RTK弄出来的里程计。
    同时，轨迹跟踪的方式也随即改变了，原版ego用的是基于加速度前馈的PD控制，然后发期望姿态来控制飞机，而现在是直接发期望的局部速度给飞机。这样有几点好处，一个是高速情况下轨迹跟踪效果会更好（你可以试试3~4m/s），另一个是下一点要讲的二维码降落。
    实际上，陶老师在接项目的时候要求有二维码降落，作为一个额外功能。所以康少写了一个视觉伺服降落，那个玩意输出的是一个速度。你会想着如果是原版ego的话，你把这个速度弄进控制器里不也行？但是事实是，效果十分狗屎，不要问我为什么，我和康少测了两个月。然后如果用px4_crtl_v2的话，只要坐标系对齐了，连参数都不用调，美滋滋。
    然后就是硬件方面的工作了，用了超杰画的小电脑作为机载电脑（主要是为了测试小电脑、然后减重还有就是阿木那破烂电脑会关机），然后飞控用的是雷迅的x7+（微贵，被陶老师吐槽了），然后两个d435。

3、如何编译？

    在你安装完全套ego所需的环境之后，还要再安装一点东西
    sudo apt install ros-noetic-mavros-extras （这一条是因为要把里程计信息给px4处理，这是必须的）
    sudo apt install python3-pip
    pip3 install pyrealsense2（这两条是因为康少写的代码是Python的，要装py环境）
    然后就可以catkin_make了

4、如何测试一下vins？

    在你弄完vins外参内参啥的之后，你需要先测试一下vins好不好用
    你需要运行shfile文件里面的test.sh脚本，然后小晃一下飞机，看看vins飘不飘
    如果没问题，直接就可以起飞了，在此之前你需要确保在QGC地面站里面EKF2_AID_MASK参数为24，然后px4固件版本为1.13版本（用1.14版本直接报废，你要是想炸机也可以试试）
    这个时候你有了px4的local position estimate，可以直接把摇杆拨到位置模式，如果它可以悬停住了，就成功了。

5、正式运行ego

    在你运行之前，需要确认一下参数，这份代码默认是跑waypoint模式，不是rviz打点模式，所以你需要先更改你想要的waypoint。然后默认起飞高度是2m。
    首先打开你的遥控器，然后拨到位置模式
    然后运行rspx4.sh
    运行rosrun ego_planner Auto_land.py
    然后这个时候等rviz打开，晃一下飞机，看看飘没飘，如果没飘就可以继续了
    运行rosrun ego_planner px4_ctrl_v2
    然后遥控器内八解锁，飞机起桨，然后摇杆拨到offboard模式（要之前在qgc设置好），然后飞机就能正常飞了，如果飞行途中看到二维码的话，是会降落的。

6、紧急情况

    如果飞机看起来运行的不对，就马上拨到position模式，如果此时vins没飘，你是可以控制的，如果还不行，就直接自稳吧
    如果你看到rviz里面飞机有点跟不上那个轨迹，建议直接拨position模式，这个问题的原因很多，有可能是电脑发热了，导致里程计卡住了一下，至于为什么，我也不知道。
